Instructions

Terminology:
Sequence The input may be TCW-assembled transcripts, transcript sequences, proteins or genes.
Conditions A condition may be a tissue type, treatment, etc.
AnnoDB A database in FASTA format against which the sTCW sequences were searched against for annotation. The search used BLAST or DIAMOND.
DB hit An alignment of an entry from an annoDB (e.g. UniProt) to an sequence. A given sequence can hit an annoDB multiple times (have alignments to multiple entries) and it can hit multiple different annoDBs (have alignments in more than one annoDB).
The following are characterizations assigned to DB hits
Best Bits (BS/BITS) The hit with the best bitscore. If there is a tie, a SP (SwissProt) or TR (TrEMBL) takes precedence. If there is still a tie, the best e-value takes precedence.
Best Anno (AN/ANNO) The hit with the best bitscore from the list of informative descriptions, where an informative description is one that does not contain phrases such as "uncharacterized protein". It also looks for names such as "Os02g0768200 protein", though this means it sometimes gets it wrong, e.g "B1 protein" is detected as non-informative as it has the general composition of a name. (See "Special Case" below).
Best with GO (WG) From a list sorted on bitscore, the first one with at least one GO ID assigned.
Rank The rank is assigned based on the bitscore sort followed by the e-value sort, i.e. when the bitscore is tied, the order is based on the e-value. Note that the Blast and Diamond default order is the e-value sort followed by the bitscore.
NOTE: The bitscore takes precedence over the e-value since the e-value takes into account the size of the database, wherease the bitscore will be the same regardless of the size of the database.

Special Case - Best Anno (AN/ANNO): The database may having been built with the "SwissProt precedence" option set, in which case, this will be stated at the end of the Overview panel. When this is selected, a SwissProt hit will be selected if its bitscore is sufficiently close the non-SP best annotation.

Detailed Help is provided on each page; the following provides an overview of what is available.

General

>>Instructions - this page

>>Overview - Summary of everything in database.

>>Decimal Display - Specify how to display decimal numbers and highlight p-values.

>>Find hits - Search a sequence against the sequences in the database or a user selected protein database. This requires the Blast and optional Diamond paths to be in the HOSTS.cfg file.

Sequences

>>Show All - A table of all sequences in the database.
  • To view a sequence from the table, select the sequence and then "View Selected Sequence".
  • Change the columns in the table using "Columns" under "Sequence.
>>Columns - Menu of columns for the sequence table.
  • Select or deselect columns.
  • The columns are not immediately changed for existing tables. Either select "Refresh All Existing Tables" or redisplay the table (listed on the left panel) and select "Refresh columns".
>>Filters - A menu with specific filters for sequences.
  • Select one or more filters, then select "View Filtered Sequences". The sequences that pass all filters will be displayed in a table.
  • If you are not seeing the columns you desire, go to the "Columns", change the columns, come back to the table (it will be listed on the left), and click "Refresh Columns".
>>Results - List all Filter query results.
  • All queries that have been performed will be listed in a table.
  • Selecting a label will redisplay it on the right, or select it from this Results table.
  • To remove a label, select the 'x' beside it on the left panel, or remove it from the Results table.

Basic Queries

>>Sequences - Search on Seq ID, TCW Remarks or User Remarks.
  • This builds a table from the results of the selected filters.
  • The resulting sequences can be shown as "View Selected Sequence".
  • Columns can be changed with "Select Columns".
>>AnnoDB Hits - Search on the Hit Id (e.g. UniProt ID) or description.
  • It works like the basic Sequences.
>>GO Annotation - Search on the GO ID or description.
  • It works like the basic Sequences.
