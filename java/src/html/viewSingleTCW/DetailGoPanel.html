Seq Detail GOs

See GO Help for explanation of terms.

First four columns of table:

  • GO ID: The GO identifier.
  • GO Name: The term name from the GO database for the GO ID.
  • Ont: Abbreviation of the ontology (top level): BP is biological_process, CC is cellular_component, MF is molecular_function.
  • Level: A GO ID can be at multiple levels in the tree. This is its maximum level.
The remaining columns depends on the GO... pull-down view:
  • Assigned GOs for all hits
    • #:EvC: Each hit-GO association has a EvC (evidence code). This list provides the number of hits with a given EvC for the GO.
    • Best Hit ID and E-value: Of all the hits for the sequence with the GO, this is the Hit ID with its best E-value.
  • All GOs for all hits
    • Assigned: If yes, then there is at least one hit-GO pair that is assigned.
    • Best Hit ID and E-value: Of all the hits for the sequence with the GO, this is the Hit ID with its best E-value. NOTE: The Show... option will show all Hit ID/E-values for a selected GO.
  • Assigned GOs for selected hit
    • EvC: When a GO is assigned to a hit, it is assigned an evidence code.
  • All GOs for selected hit
    • EvC: If this is in parenthesis, it has been inherited.

GO Options - second row of buttons

Selected: The four GO options produce exactly the same output as their counterparts on Basic GO.
  • Show...
    • Seq Hits - assigned  Hits to the current sequence that have the GO assigned.
       
    • Seq Hits - inherited  Hits to the current sequence that have the GO inherited. This can take a few minutes for a high level GO.
       
    • GO - Neighborhood with relations The immediate parents and children of the GO ID. The relation of 'is_a' or 'part_of' is shown for this option. NOTE: same note as for descendants.
       
    • GO - Ancestors   All ancestors of the GO ID sorted by level.
       
    • GO - Descendants   All descendants of the GO ID sorted by level. NOTE: if a descendant GO does not have any assigned or inherited hits, it is not in the database so is not shown.
       
    • GO - Ancestor path table   The same GOs as in the Ancestor List, but shows the paths in table form. If all paths have the same GOs at the top and bottom of the graph, they are only shown once and the row will be gray. NOTE: Occasionally, this query may be slow when there are many paths for the GO; some GOs have over 1000 paths.
  • Export...   The exact same options as above, but the results are written to file. When an "*" is at the end of the option, this indicates that there will be a choice between writing All Info or ID only. For example, if the Hits - assigned* is selected and ID only is then selected in the popup window, the HitIDs will be written to file; this is useful for loading in the Basic Hit panel to view.
Table:
  • Show...
    • Table columns  A copy of the table in a popup window.
       
    • GO sets: (Assigned GOs only) Different sets of GOs may be assigned to the different proteins that hit the sequence. This shows how they group. For example: 
                Sets   1   2  #Sets
   #GOs in set   1   5 
    GO:0005829       X   1
    GO:0080045       X   1
    GO:0080043       X   1
    GO:0080046       X   1
    GO:0080044       X   1
    GO:0016758   X       1
#Hits with set  25   1
      The first set only has one GO, and 25 hits were assigned the GO. The second set has 5 GOs, but only one hit was assigned the 5 GOs.
  • Export...
    • Table columns  The table is written to file in a tab-delimited format.
    • GO IDs only  The GO IDs are written to file, one per line.
