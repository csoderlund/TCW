Basic Filters for Sequence

This provides some basic sequence searches; for more involved searches, use the Filter bullet on the left panel.

Search

Choose one of four search modes and enter a string to search on.

Seq IDSequence name. Only one name is allowed unless loaded from file.
Orig ID(1) Original name if Seq ID != Orig ID (2) Longest read of contig if assembled, (3) no query if neither of the previous.
TCWTCW Remarks are entered when the database is built; they provide information on the ORFs, and if the sequence has hits in multiple frames.
UserUser Remarks may be associated with each sequence (entered with runSingleTCW).
Search rules
  • By default, the search is for the substring anywhere in the ID or remark.
  • To force it to match at the beginning of the string, enter a '%' at the end of the substring, e.g. 'mitochondrial%' to match everything that starts with 'mitochondrial'. To force it to match at the end of the string, enter a '%' at the beginning of the string. NOTE: since '%' is a special character, it cannot be in a search string as a literal.
Load File:
  • Select the search mode: Seq ID, Orig ID, TCW or User.
  • The file should contain one search item per line. This performs an exact search.
  • For Seq ID and Orig ID, there can be additional information on a line; only the first word will be taken as an ID.

Results

Select one or more filters, the select one of the following:
BUILDBuilds a new table.
ADDAdds any new rows to the existing table.
Columns: This will bring up a panel of columns, which can be selected or deselected, and the results will immediately be changed in the table after Accept.

A column can be moved by selecting the heading and moving it. Click the column heading to sort it.

Columns

TCW RemarkAssigned by TCW during runSingleTCW ORF computation. See last section.
User RemarkUploaded by the user with runSingleTCW.
CountTotal count from all conditions/libraries.
Orig ID or LongestSee Search above.
Best HitIDThe HitID of the hit with the best e-value.

TCW Remark Legend

ORF=HitThe ORF has the same coordinates as the hit.
ORF=Hit+The ORF has the same coordinates as the hit and Start and Stop codons.
ORF>HitThe Hit is contained in the ORF.
ORF<HitThe ORF is contained in the Hit.
ORF~HitThe ORF coordinates overlap the Hit.
$The dollar sign may be appended to any of the ORF remarks above, which indicates that the ORF has a Hit, is the longest, and has the best Markov score (there will not be a ORF!Lg or ORF!Mk).
ORF!LgThe ORF is not the longest in comparison with the other 5 frames.
ORF!MkThe ORF does not have the best Markov score in comparison with the other 5 frames.
!5!3No Start or Stop codon.
!5No Start codon.
!3No Stop codon.
ORF!HitThe ORF frame is not the same as the Hit frame.
ORF>=9NsThere are >=9Ns in the ORF.
ORF:ANTypically, the BS (Best Bits) hit is used. But if the BS is a nucleotide hit, then the AN (Best Annotation) hit is used.
Multi-frameHits to multiple different frames
StopHitnStops in best hit, where n is the number of Stops found.
