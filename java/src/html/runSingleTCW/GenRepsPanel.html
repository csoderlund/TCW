Define Replicates

The table will be populated with the Associated Counts table on the main panel. The three columns are as follows:

SeqID The name of the sequence dataset that the count file is associated with.
Column name from file This column lists the replicate identifiers (column headers) which were found in the Count File. Each replicate will have a unique name, for example "MyLib1, MyLib2, MyLib3".
Condition This is the column to define condition (e.g. tissue, treatment) for each replicate set. If your replicate identifier (Column name from file) are of the form name+number, as in the root1, root2, root3 example, then the condition (e.g. root) will be automatically entered and you will not need to edit the name. Otherwise you will need to make up suitable condition names and edit the corresponding rows. Make sure that these condition name is unique and is not the same as any SeqIDs. To edit the 3rd column, put your cursor in the cell you want to edit, hit the backspace keep and the cell will turn white indicating you can edit it.

When the table is correct, press Keep, which will update the Associated Counts table on the main panel.

Note: If you are creating multiple singleTCW databases to compare with multiTCW, it is extremely important that you use the exact same condition names for equivalent conditions, e.g. if you have a maize and rice database where they both have 'leaf' and 'root', give them the same name so they will correspond in multiTCW.
