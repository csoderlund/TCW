Sequence Table

This table can be viewed from (1) Sample List "Sequences", (2) Filter "Sequence", or (3) selecting a row in the cluster, pairs or hit table.

First row of functions at the top

The bottom panel can be switch between Table and the results of any of the "Align:" buttons.

Aligned (selected): If none are selected, the default selection is all rows.

  • Pairwise...
    AA for each pair Perform pairwise alignment for all pairs of amino acid sequences from the selected sequences.
    NT for each pairNT-mTCW only Perform pairwise alignment for all pairs of nucleotide sequences from the selected sequences (.
    AA,NT,CDS for one pair NT-mTCW only; 2 seqs Align the selected pair using the amino acid sequence, CDS sequence, and nucleotide sequences. The resulting panel will have these 3 alignments.
    5UTR, CDS, 3UTR for one pair NT-mTCW only; 2 seqs Align these three regions separately for the pair. The resulting panel will have these 3 alignments.
    AA to sequence best hit The best hit for each sequence will be aligned to it.
    AA to cluster best hit For Cluster only The assigned cluster best hit is aligned to each sequence in the cluster.

    The Pairwise options can result in many alignments, e.g. if 20 sequences are selected, 190 alignments will be computed; watch progress in lower left corner of viewMultiTCW.
     
  • MSAdb (Cluster set of sequences only) The MAFFT-AA alignment has been stored in the database, so can be displayed without running the program. However, this option does not produce the alignment or score files, where the MSA... option does.
     
  • MSA...
    • MUSCLE-AA: Run MUSCLE to build a multiple alignment for all amino acid sequences in the table or for the selected set. This can be very time consuming when run on many sequences. The command executed is: 
      /Ext/linux/muscle/muscle -in ResultAlign/inSeq.fa -out ResultAlign/outAln.fa
      You can view the results along with the scores in the ResultAlign directory. The first score is the average of the column sums from the sum-of-pairs. The second is the trident method, run as follows (use -g to output global score only): 
      /Ext/linux/mstatX/mstatx -s trident -i ResultAlign/outAln.fa -o ResultAlign/score2.txt
    • MAFFT-AA: Same as above. The command executed is: 
      /Ext/linux/mafft/mafft.bat --auto --reorder --thread N ResultAlign/inSeq.fa
      Where N is the number of available threads (processors/CPUs).

    • MAFFT-CDS: (NT-mTCW only) Same as above but with the CDS sequences

    • MAFFT-NT: (NT-mTCW only) Same as above but with the NT sequences

    On Mac, replace Ext/linux with Ext/mac.

  • Table...
    • Show Column Stats: Popup a table of statistics for the numeric columns in the table. Note: if you Export the table and compute the statistics, their results will not be as precise.
       
    • Copy Table: Copies the table to the clipboard either the entire table.
       
    • Export Table: Exports the table to file in tab delimited file.
       
    • Export AA sequences:Export the translated CDS sequences..
       
    • Export CDS sequences:Export the CDS sequences.
       
    • Export NT sequences:Export the full nucleotide sequences.
       
    • Export all GOs: output all assigned and inherited GOs for all seq-hit pairs for all sequences in the table. The first three are filters::
      1. GO Level:
        1. Level 1 is the three domains. Level 2 are the GO terms directly beneath a domain, etc (see GO annotations 'GO Help' for more explanation).
        2. If you leave this blank, it will output all GOs.
        3. If you just want a couple of levels, e.g. 2 and 3, first export level 2, then export level 3 to the same file and use the append option (there will be a pop-up asking whether you want to append).
      2. Any domain: This toggles through the three main domains. If you select one, it will only export the GOs in that domain.
      3. E-value: If you leave this blank, it will ignore the E-value. Otherwise, the sequence must have at least one seq-hit with E-value≤ this value.
      4. File Columns:
        1. Verbose: outputs columns GOterm #Seqs Level Term_type Description
        2. Count only: only outputs the GOterm and #Seqs.
          This can be used as input to input to REVIGO (http://revigo.irb.hr)

Second row of functions at the top

  • Details For the selected sequence: show counts, TPM values, and annotations.
     
  • Pairs For the selected sequence(s): Show the table of pairs containing at least one of the selected sequences.
     
  • Clusters For the selected sequence: Show the table of clusters containing the sequence.
     
  • Copy... For the selected sequence: Copies the seqID, AA, NT or CDS sequence, Best hitID, Best description or Best hit sequence.
     
  • Clear Clears the selected sequences.

Columns

  • Columns can be added/removed using the Select Columns button below the table.
  • Columns can also be rearranged (by dragging the headers) and sorted (by clicking the headers).
  • Put the mouse over a column heading in the Select Columns interface, and a full explanation of the column is shown in the lower left corner.
