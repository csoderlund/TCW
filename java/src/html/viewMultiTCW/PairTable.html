Pair Table

There is a pair for a given two sequences if they have a NT (nucleotide) of AA (amino acid) hit, which are computed when the mTCWdb is created. There are only coding statistics and KaKs values for pairs in clusters (it depends how the database was built -- these values may only exist for certain cluster sets).

Functions at the top

Sequences Select a pair followed by this function to view the two sequences in the sequence table, where the alignment can be viewed.

Clusters Select a pair followed by this function to view the clusters for the two sequences.

Copy... For the selected pair, copy to clipboard a column (SeqID1, SeqID2, HitID or Hit Description).

Table...

  • Show Columns Stats: Popup a table of statistics for the numeric columns in the table.
     
  • Show Table Stats: (NT-mTCW only) Popup a table of the coding and KaKs statistics for the pairs in the table. This only works for pairs that have coding sequence statistics.
     
  • Copy Table: Copies the table to the clipboard either the entire table.
     
  • Export Table: Exports the table to file.
     
  • Export both AA/NT sequences: Exports both AA (amino acid) or NT (nucleotide) sequences for all pairs in the table.
     
  • Export 1st AA/NT sequences: Exports the 1st AA or NT sequence of the pair for all pairs in the table.
     
  • Export Pair GOs: list GOs associated with the pair sequences in the table. If either sequence in the pair has a hit with a GO the passes the below filters, the pair is counted as having the GO. NOTE: it does NOT use the Pair "Best Anno Hit". The first three are filters::
    1. GO Level:
      1. Level 1 is the three ontologies (top level). Level 2 are the GO terms directly beneath the top level, etc.
      2. If you leave this blank, it will output all GOs.
      3. If you just want a couple of levels, e.g. 2 and 3, first export level 2, then export level 3 to the same file and use the append option (there will be a pop-up asking whether you want to append).
    2. Any Ontology: This toggles through the three main ontologies. If you select one, it will only export the GOs in that ontology.
    3. E-value: If you leave this blank, it will ignore the E-value. Otherwise, at least one sequence in the cluster must have a seq-hit E-value≤ this value.
    4. Count limit: If a GO term has less than this count of GO-seq assignments, it is not written to file.
    5. File Columns:
      1. Verbose: outputs columns (GO is the column for the ontology abbreviation) GO-ID #Pair Level GO GO-Name
      2. Desc/Count: outputs the name and count by ontology.
        This can be used in Excel to create graphs.
      3. GO/Count: only outputs the GO ID and #Pairs.
        This can be used as input to input to REVIGO (http://revigo.irb.hr)

Columns

  • Put the mouse over a column name in the Select Columns interface, and a full explanation of the column is shown in the lower left corner.
  • Columns can be added/removed using the Select Columns button below the table.
  • Columns can also be rearranged (by dragging the headers) and sorted (by clicking the headers).
  • One or more pair rows can be selected using mouse click, shift-click, or ctrl-click.
The columns available depend on whether the mTCW database was build with nucleotide or protein sequences. The sections are as follows, where most columns are adequately explain with the mouse-over, except the following:
General 
  HitIDThe best shared hit from the pair (see the online documentation for a description of how this is computed).
The HitID "*NoShare" indicates the pair does not share any description.
The HitID "*Novel" indicates that neither sequence of the pair has any annotation.
  PCCPearson Correlation Coefficient of the TPM values. This only has values>0 if the datasets have matching condition names.
HitResults from protein and DNA sequence self-blast (if available). All columns are directly from Blast or Diamond output except the 'cov' columns.
  Cov1, Cov2The percent coverage, which is computed from the 'align' divided by sequence length. The Cov (percent coverage) can be >1; this happens when the alignment covers the whole sequence with gaps.
KaKs (NT-mTCW only) Ka is the synonymous substitution rate. Ks is the nonsynonymous substitution rate. KaKs is Ka/Ks. The p-value applies to the KaKs value.
