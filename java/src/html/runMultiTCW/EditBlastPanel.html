Search Settings

mTCW requires the AA hit file, which is the result of a file of sequences in the databases being searched against itself. The NT hit file is optional. The respective hit files are used to create pairs in the database, and the pairs are used to compute the BBH and Closure clusters. For orthoMCL, the pairs are read from the AA blast file.
  • If the hit file (hitsAA.tab) does not exist: the main panel will say "<Search-program> AA file", where <Search-program> may be Blast, Diamond or Usearch. Before executing "Run Search", you can set the search program (see www.agcol.arizona.edu/tcw/doc/Search.html), and you can change the default parameters.

  • Important. If the hit file DOES exist, then the main panel will say "Use existing file" and will not let you re-create the hit file. To generate a different hit file, you need to remove the existing one, which can be done from the main panel with the "Remove..." button. Alternatively, you can rename the existing hit file.

These same instruction apply to the NT blast file, but you cannot select the search program as only Blastn can be used for the NT self-blast. This option is NOT available if any of your input sTCWdbs are created from protein sequences.

Running the search yourself

If you want to run the search on a different machine (e.g. computed on an HPC cluster):
  1. When the database is built, it creates a sub-directory projcmp/<project name>/hitResults which contains the file of proteins sequences (combineAA.fasta).
  2. Use the fasta file as input to blast and have it produce the tabular output.
  3. Put the blast tabular formated file in the hitResults directory with the name hitsAA.tab.
The same instructions apply generating the nucleotide blast file, where the input would be combineNT.fasta to produce the file hitsNT.tab; this only applies if none of the input sTCWdbs are created from protein sequences.
